### P0-1. Schema 语法错误 — password_resets 建表失败

**位置**：§5.1 Schema 第1062-1071行

**问题**：`CREATE INDEX` 语句被写在了 `CREATE TABLE` 的括号内部，SQL 语法错误，首次访问时建表会直接失败。

**当前代码**：
```sql
CREATE TABLE IF NOT EXISTS password_resets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT NOT NULL,
    token TEXT NOT NULL UNIQUE,
    expires_at TEXT NOT NULL,
    used INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime')),
    CREATE INDEX IF NOT EXISTS idx_reset_token ON password_resets(token, used, expires_at)  -- ❌ 错误位置
);
```

**修正方案**：将 `CREATE INDEX` 移到 `CREATE TABLE` 括号外面：
```sql
CREATE TABLE IF NOT EXISTS password_resets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT NOT NULL,
    token TEXT NOT NULL UNIQUE,
    expires_at TEXT NOT NULL,
    used INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now','localtime'))
);
CREATE INDEX IF NOT EXISTS idx_reset_token ON password_resets(token, used, expires_at);
```

> ⚠️ 同样的问题也存在于 §4.1 config.php 的 v4 迁移代码中，需同步修正。

### P0-7. v5 迁移存在外键违规风险

**位置**：§4.1 config.php v5 迁移代码，§5.5 迁移机制

**问题**：`COALESCE(ue.preset_exam_id, 0)` 会产生 `exam_id=0` 的记录，而 exams 表里没有 id=0 的记录。由于 `PRAGMA foreign_keys = ON`，这些记录会因外键约束失败而被拒绝插入，导致迁移中断、数据丢失。

**当前代码**：
```php
"INSERT INTO scores_new (exam_id, user_id, subject, score, full_score, grade_rank, class_rank, created_at)
    SELECT
        COALESCE(ue.preset_exam_id, 0) AS exam_id,  -- ❌ 可能产生0
        ..."
```

**修正方案**：
```php
// 方案A：跳过无法映射的旧数据（推荐）
"INSERT INTO scores_new (exam_id, user_id, subject, score, full_score, grade_rank, class_rank, created_at)
    SELECT
        ue.preset_exam_id AS exam_id,
        ...
    FROM scores s
    JOIN user_exams ue ON s.user_exam_id = ue.id
    WHERE ue.preset_exam_id IS NOT NULL",

// 方案B：迁移前先关闭外键检查（不推荐，数据一致性风险）
// "PRAGMA foreign_keys = OFF",
// ... 执行迁移 ...
// "PRAGMA foreign_keys = ON",
```

> ⚠️ goal_results 的迁移也有同样问题，需同步修正。

## 四、其他发现

### 4-1. 开发计划时间线过时

§13 开发计划仍提到"预设考试"，应改为"考试管理（按规则创建/约束校验）"。开发优先级 P0 也需同步更新。

### 4-2. 缺少 404 页面

前端路由没有定义 `/:pathMatch(.*)*` 通配路由，未匹配路径会白屏。需新增 `NotFoundView.vue`。

### 4-3. 缺少路由守卫实现规格

文档定义了 `meta: { auth: true, admin: true, guest: true, public: true }`，但没有给出路由守卫（`router.beforeEach`）的具体实现逻辑。

### 4-4. 前端目录缺少 `api/types.ts`

项目目录结构中 `src/api/` 只有笼统说明，没有列出具体的 API 模块文件和类型定义文件。

### 4-5. 前端缺少 `.env` 环境变量

开发环境和生产环境的 API 地址配置方式未说明。建议：
```
# .env.development
VITE_API_BASE_URL=http://rs.cn/api

# .env.production
VITE_API_BASE_URL=/api
```
